#Анлиз A/B-теста нового рекомендательного алгоритма

## О проекте

Наша команда разработала новый алгоритм рекомендаций для ленты новостей. Его задача — показывать пользователям более интересный контент, чтобы они чаще ставили лайки. Прежде чем внедрять изменения для всех пользователей, мы провели A/B-тест.

### Бизнес-задача
Увеличить вовлечённость пользователей за счёт более релевантных рекомендаций. Если новый алгоритм работает лучше старого — внедряем. Если нет — дорабатываем.

---

## Что сделали

### Дизайн эксперимента
- **Длительность:** 7 дней
- **Участники:** ~20 000 пользователей
- **Контрольная группа (50.4%):** старый алгоритм
- **Тестовая группа (49.6%):** новый алгоритм

### Главная метрика
**CTR (Click-Through Rate)** — доля постов, которым пользователь поставил лайк, от всех просмотренных.

> Почему CTR? Лайк — это явное действие, которое показывает: «мне интересно». В отличие от простого просмотра, лайк нельзя поставить случайно.

---

## Что обнаружили

### Основной результат
Новый алгоритм **не улучшил**, а незначительно **ухудшил** вовлечённость:

| Группа | Средний CTR |
|--------|-------------|
| Старый алгоритм | 21.68% |
| Новый алгоритм | 21.61% |
| Разница | −0.07 п.п. |

### Важный нюанс
Новый алгоритм повлиял на пользователей неравномерно:
- **Часть аудитории** стала ставить лайки чаще
- **Другая часть** — наоборот, реже
- В итоге средний показатель почти не изменился, но разброс вырос

Это говорит о том, что алгоритм нужно настраивать под разные сегменты пользователей, а не применять одинаково для всех.

---

## Как проверяли результаты

Мы использовали несколько статистических методов, чтобы исключить случайность:

| Метод | Что показал |
|-------|-------------|
| T-тест Уэлча | Различий нет |
| Тест Манна-Уитни | Различия есть ✅ |
| Сглаженный CTR + тесты | Различия есть ✅ |
| Пуассоновский бутстреп | Различия есть ✅ |

**Вывод:** большинство надёжных тестов подтверждают, что разница между группами реальна, а не случайна.

---

## Рекомендация

### ❌ Не внедрять алгоритм в текущем виде

**Почему:**
1. Средний CTR снизился (хоть и незначительно)
2. Алгоритм работает нестабильно — помогает одним, мешает другим

**Что делать дальше:**
1. Проанализировать, на каких сегментах алгоритм сработал хорошо
2. Доработать модель с учётом разных типов пользователей
3. Провести повторный A/B-тест
